【RN】React Native环境配置及踩过的坑

xiaozhangtx 2/19/2023 RN

由于最近要有客户端开发的需求，所以需要搭建开发环境，看了React Native (opens new window)的中文文档，我认为在windows11上配置环境是小菜一碟，但是结束是我太天真了。

1 安装依赖

React Native的最新版本已经是0.71，所以需要下载Node 的版本应大于等于 14，JDK要大于11

1.1 Node安装

具体安装教程可以参考node.js安装及环境配置超详细教程【Windows系统安装包方式】 (opens new window)

如果电脑上已经安装了其它node版本或者想方便管理各个版本的node这里推荐使用nvm (opens new window)这个工具，nvm的安装和使用教程可以参考nvm for windows 下载、安装及使用 (opens new window)

为了加快npm下载依赖的速度，推荐使用淘宝镜像，但文档不建议使用cnpm,具体配置如下：

npx nrm use taobao
1

1.2 Java JDK11安装

jdk安装教程可以参考jdk11下载与安装教程（win10）超详细 (opens new window)

2 配置开发环境

2.1 安装 Android Studio

首先下载和安装 Android Studio (opens new window)，国内用户可能无法打开官方链接，请自行使用搜索引擎搜索可用的下载链接。这里推荐的安装教程为Android Studio安装及启动教程 (opens new window)

注意点

模拟器最好选择API Level 31 image

2.2 安装 Android SDK

Android Studio 默认会安装最新版本的 Android SDK。目前编译 React Native 应用需要的是Android 12 (S)版本的 SDK（注意 SDK 版本不等于终端系统版本，RN 目前支持 android 5 以上设备）。你可以在 Android Studio 的 SDK Manager 中选择安装各版本的 SDK。安装教程可参考Android sdk下载安装配置教程 (opens new window)

注意点

  • 安装SDK的文件目录最好放到其它盘，文件比较大，并且文件目录最好不要出现中文
  • 版本一定要安装对
  • 确保选中了 React Native 所必须的33.0.0版本。你可以同时安装多个其他版本。
  • 确保勾选了下面这些组件（重申你必须使用稳定的代理软件，否则可能都看不到这个界面）：Android SDKPlatform 31和Intel x86 Atom_64 System Image（官方模拟器镜像文件，使用非官方模拟器不需要安装此组件)
  • 安装时间有点长，建议耐心等待，或者学会使用代理

3 创建新项目

注意点

如果你之前全局安装过旧的react-native-cli命令行工具，请使用npm uninstall -g react-native-cli卸载掉它以避免一些冲突：

npm uninstall -g react-native-cli @react-native-community/cli
1
  • 请不要在目录、文件名中使用中文、空格等特殊符号。请不要单独使用常见的关键字作为项目名（如 class, native, new, package 等等）。请不要使用与核心模块同名的项目名（如 react, react-native 等）。
  • 请不要在某些权限敏感的目录例如 System32 目录中 init 项目！会有各种权限限制导致不能运行！
  • 请不要使用一些移植的终端环境，例如git bash或mingw等等，这些在 windows 下可能导致找不到环境变量。请使用系统自带的命令行（CMD 或 powershell）运行。

3.1 创建项目

npx react-native init 你的项目名称
1

3.2 开启模拟器

模拟器最好选择API Level 31 image

3.3 运行项目

 npx react-native run-android
1

编译这个过程视情况而定，一般时间比较久，我大约等了3个多小时，才编译成功 最终编译成功可以看到

4 我遇到的问题

4.1 run-android下载包很慢

解决方法： 修改了android->build.gradle文件中repositories

    repositories {
//        google()
        maven { url 'https://maven.aliyun.com/repository/jcenter' }
        maven { url 'https://maven.aliyun.com/repository/google' }
        mavenCentral()
    }

修改后网速稍微快了那么一点点！

4.2 环境配置

在为运行 run-android时，给我报环境问题 我但是就懵逼了，后面使用了

 npx react-native doctor
1

这个指令发现了问题所在，原来是我的版本问题，然后fix就解决了

4.3 java版本问题

由于我的JAVA_HOME是java8，所以在编译的时候给我报了版本不支持的问题 但是我设置了我的Project Structure为java 11但是没有用，最后我尝试修改了android->gradle.properties，加了一句

org.gradle.java.home=E:/JDK11 //你jdk11的地址
1

5 总结

配置React Native 太麻了！
